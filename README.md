# AI Incident Detection System

A deep learning project focused on predicting AI-related incident risk by combining **code vulnerability detection** with **incident risk analysis**.

This repository documents **my contribution** to a broader team project that explored how software vulnerabilities and system-level risk indicators can be used to estimate the likelihood of AI failures.

## Team Project Reference
This repository reflects my individual contribution to a collaborative project.

Original team repository: [AI-Incident-Detection-System](https://github.com/aayushi2812/AI-Incident-Detection-System)

## Project Overview
AI incidents can arise from multiple sources, including insecure code, weak software reliability, and system-level risk factors. The broader project proposed a multi-model approach for analyzing these risks and estimating incident probability.

The full system included:
- **Model A -- Code Vulnerability Detection**
- **Model B -- Data Quality Anomaly Detection**
- **Model C -- Incident Risk Classifier**

## My Contribution
In this team project, my main contributions were in:

### Model A -- Code Vulnerability Detection
- Developed code for the **code vulnerability detection** component
- Contributed to implementation related to identifying vulnerability patterns in source code
- Supported the analysis of code-based risk signals for downstream incident prediction

### Model C -- Incident Risk Classifier
- Performed analysis for the **incident risk classifier**
- Developed code for **incident risk distribution**
- Implemented and analyzed the **calibration curve** to evaluate prediction reliability
